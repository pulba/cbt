Penjelasan Detail Fitur: E. Portal Siswa (Peserta Ujian)
Bagian ini adalah front-end yang langsung bersentuhan dengan Siswa saat mengerjakan ujian. Memiliki sistem perhitungan waktu server (menghindari kecurangan manipulasi jam komputer), dan pencatatan acak (randomisasi) butir soal individual untuk tiap siswa.

1. Dashboard Ujian & Distribusi Token (Tes_dashboard.php)
Fitur pendaratan pertama setelah Siswa menekan login, menampilkan ujian apa saja yang sedang menjadwalkan siswa tersebut (terkait dengan status Grup nya).

Logika & Fungsi Berjalan:

Validasi Ketersediaan Ujian (get_datatable):
Mengambil data seluruh jadwal tes dari tabel cbt_tesgrup berdasar Grup/Kelas siswa yang login.
Hanya menampilkan tombol Kerjakan jika tes_begin_time sudah masuk, dan membuangnya dari daftar jika tes_end_time telah lewat.
Konfirmasi Tes & Sistem Token (konfirmasi_test & mulai_tes):
Saat siswa menekan Kerjakan, akan muncul layar persetujuan (Mengetik Token / Baca Instruksi Awal Ujian).
Jika wajib token, di mulai_tes() dilakukan validasi: Apakah token yang dimasukkan masih terdaftar dan berumur valid? Apakah token ini ditujukan untuk Semua Ujian atau eksklusif untuk Ujian ini saja?
Proses Cetak Biru Acak Soal (The Randomization Engine):
Ini adalah "Inti Perakitan" seluruh Ujian CBT.
Saat token valid, mulai_tes() memulai Database Transaction.
Sistem memasukkan user_id ke tabel rekam pengerjaan cbt_tes_user.
Sistem kemudian membaca "Rancangan/Blueprint Ujian" (berapa jumlah soal per topik, apakah acak nomor soal, dan apakah acak opsi pilihan ganda).
Melakukan looping query pengambilan soal secara acak ke Bank Soal langsung (dengan perintah query custom Acak dari DB jika konfigurasi disetel ON).
Soal-soal yang terambil secara random itu kemudian dicatat permanen satu-per-satu ke Tabel Log Siswa (cbt_tes_soal) dengan urutan tersendiri per siswa (berbeda antara Budi dan Andi).
Sama halnya dengan Pilihan Jawaban (A, B, C, D, E). Mesin mengacaknya, lalu menyimpannya dalam urutan urut 1 sampai 5 ke cbt_tes_soal_jawaban per siswa.
Hal ini menjamin: Pilihan A pada Budi bisa jadi menjadi Pilihan C pada Andi.
2. Antarmuka Pengerjaan CBT (Tes_kerjakan.php)
Controller yang mengontrol antarmuka ujian siswa secara single-page-app view, mencegah relokasi tab berlebihan.

Logika & Fungsi Berjalan:

Perhitungan Waktu Hitung Mundur / Countdown (index):
Sistem tidak percaya dengan waktu komputer/HP siswa.
Waktu start pertama kali di-stamp di database MySQL server, yakni tesuser_creation_time. Waktu habis adalah waktu pengerjaan (Misal 120 Menit).
PHP menghitung sisa detik (detik_total - detik_berjalan) berdasarkan relasi Server Time $tanggal_diff = $tanggal_tes->diff($tanggal_saat_ini). Sehingga ketika Siswa me-refresh, atau menutup jendela browser & login Device lain, durasi waktunya tidak mereset, namun terus berjalan dari titik rekaman asalnya. Jika hitungan di atas 0, dikirim ke frontend JS untuk divisualisasi hitung mundurnya. Bila habis, dikembalikan ke Dashboard.
Memanggil get_daftar_soal() untuk merender Kotak-Kotak Penanda Soal 1, 2, 3... dst. beserta state warnanya (Putih = Belum Dijawab, Biru = Sudah Dijawab, Kuning = Ragu-Ragu).
Format Soal via Ajax (get_soal_by_tessoal, get_soal):
HTML soal, termasuk pengkonversian link direktori Base64 Audio/Image tag di-generate utuh oleh Controller PHP, lalu disodorkan kembali ke layar siswa (DOM replace).
Fitur Audio Listening (soal_audio_play) diawasi secara rekat memproteksi jumlah putar audio (misal jatah memutar maksimal hanya dikasih 1 kali). Saat tag audio diputar, sistem kirim ajax ke update_status_audio untuk membekukan sisa pemutaran sehingga tak bisa direplay.
Auto-Save Jawaban (simpan_jawaban):
Tiap siswa memilih pilihan Ganda (Radio Button A, B, C) atau mengetik jawaban Esai / Essay Singkat, JS Trigger akan menembak AJAX ke .simpan_jawaban().
Saat masuk, waktu PHP terkini date('Y-m-d H:i:s') akan direkam sebagai tessoal_change_time. Opsi radio akan menggeser boolean soaljawaban_selected = 1 untuk pilihan terkini, dan 0 untuk yg tidak dipilih.
Jika pilihan itu benar query_jawaban->jawaban_benar==1, maka skor Poin Benar akan dicatatakan di log nilai. Sebaliknya akan dicatat poin salah. Ini meringankan beban skor di akhir ujian karena nilai kalkulasi berjalan asimetris per sentuhan siswa (Live Grading).
Ragu-Ragu (update_tes_soal_ragu):
Menggeser Flag Boolean Ragu menjadi 1 di database tessoal_ragu.
3. Hasil Ujian Selesai (Tes_hasil_detail.php)
Ketika siswa menekan Hentikan Ujian dari dalam tombol ujian, status pengerjaan ditutup (Status=4). Siswa dilempar balik.

Logika & Fungsi:

Menyajikan kembali soal satu-per-satu lewat Datatable get_datatable(), namun dibarengi dengan Kunci aslinya dan Pilihan mereka sendiri. Pilihan tertunjuk dengan label "Kunci" dan "Pilihan".
Memanggil kalkulasi nilai dari get_nilai() dengan membagi (Skor Kumulatif / Max Skors x 100), memperlihatkan perbandingan statik.
Fitur ini opsional dan murni bergantung saklar konfigurasi tes_detail_to_users dan tes_results_to_users yg dirancang di pengaturan Ujian awal (Tes_tambah.php). Jika administrator memutuskan matikan, maka portal review ini tidak bisa diakses oleh siswa.
