Penjelasan Detail Fitur: A. Manajemen Bank Soal (Modul)
Berdasarkan analisis source code pada controllers CodeIgniter yang digunakan, berikut adalah rincian mendalam mengenai cara kerja dan logika dari setiap fitur di dalam Manajemen Bank Soal.

1. Topik/Mata Pelajaran (Modul_topik.php)
Fitur ini mengelola pengelompokan soal berdasarkan mata pelajaran atau bab (Topik). Fungsi Utama & Logika:

index(): Menampilkan halaman daftar topik. Menarik data Modul dari model cbt_modul_model. Jika data Modul kosong, sistem secara otomatis membuat satu modul Default.
get_datatable(): Memproses data secara asinkron (AJAX DataTables). Mengambil data topik beserta jumlah soal yang sudah masuk ke dalam topik tersebut (menggunakan cbt_soal_model->count_by_kolom).
tambah() & edit(): Menyimpan atau mengubah topik (cbt_topik_model). Terdapat proteksi ganda:
Validasi form mencegah input kosong atau tag HTML terlarang.
Pengecekan nama topik; sistem tidak mengizinkan nama topik yang sama di dalam satu modul (mencegah duplikasi).
hapus_daftar_topik(): Menghapus topik beserta folder file media-nya.
Proteksi Hapus: Sistem mencek tabel cbt_tes_topik_set. Jika topik tersebut sudah/sedang digunakan dalam sebuah "Ujian/Tes" yang aktif, maka tombol hapus gagal dijalankan.
Jika berhasil dihapus, sistem akan menghapus data di database (cbt_topik_model->delete) sekaligus menghapus direktori khusus topik tersebut (misalnya folder uploads/topik_12) menggunakan fungsi rmdir() dan delete_files().
2. Bank Soal (Modul_soal.php & Modul_daftar.php)
Fitur untuk mengelola (tambah, edit, hapus) pertanyaan/soal ujian. Fungsi Utama & Logika:

index(): Menampilkan form soal. Di sini terdapat sistem pengecekan Akses User/Guru. Jika user yang login dibatasi hak aksesnya (berdasarkan kolom opsi1 pada tabel user), mereka hanya bisa melihat dan memilih Topik mata pelajaran yang ditugaskan kepada mereka saja.
tambah() / Simpan Soal: Logika penyimpanan soal cukup kompleks.
Parsing Gambar Otomatis: Pertanyaan (text editor) di-parsing menggunakan DOMDocument(). Jika guru memasukkan gambar dengan metode Copy-Paste (base64 image), sistem akan meng-extract base64 tersebut, menyimpannya menjadi file gambar fisik (.jpg/.png) ke folder uploads/topik_[id], lalu merubah tag HTML img src-nya mengarah ke URL gambar fisik (menghemat ruang database).
Merubah alamat base_url() menjadi placeholder [base_url] sebelum disimpan. Tujuannya agar gambar tidak error jika domain aplikasi sewaktu-waktu berubah.
Input Audio: Mengunggah file audio (.mp3) khusus untuk soal tipe "Listening".
Soal Isian Singkat: Jika tipe soal adalah Jawaban Singkat (tipe 3), sistem mewajibkan guru memasukkan Kunci Jawaban, jika kosong, sistem menolak menyimpan.
hapus_soal(): Menghapus soal hanya jika topik dari soal tersebut belum digunakan dalam Jadwal Tes yang sedang berjalan.
Modul_daftar.cetak_soal(): Memungkinkan admin/guru untuk mencetak/export seluruh soal beserta kunci jawabannya ke layar untuk di-print berdasarkan topik.
3. Kunci Jawaban (Modul_jawaban.php)
Mengelola opsi jawaban (A, B, C, D, E) untuk sebuah soal Pilihan Ganda. Fungsi Utama & Logika:

index(): Mencek terlebih dahulu tipe soal. Jawaban hanya bisa ditambahkan jika tipe soal adalah Pilihan Ganda (bukan Essai).
tambah(): Menyimpan pilihan jawaban dengan logika parsing content yang persis sama dengan soal.
Jika jawaban mengandung gambar base64, gambar tersebut diekstrak dan disimpan secara asinkron ke sub-direktori topik, supaya load ujian nanti lebih cepat.
Memberi label status Benar (1) atau Salah (0) pada teks/gambar jawaban tersebut ke tabel cbt_jawaban.
hapus_jawaban(): Jika jawaban tersebut sudah terlanjut diisi oleh siswa saat ujian (ada di tabel cbt_tes_soal_jawaban), maka sistem mencegah penghapusan jawaban untuk menghindari kerusakan pada nilai siswa.
4. Import Soal (Modul_import.php, Modul_import_word.php, Word.php)
Memasukkan banyak soal ke database secara massal tanpa harus mengetik satu-satu di web. Fungsi Utama & Logika:

Modul_import.import_file() (Format Excel):
Menggunakan library PHPExcel_IOFactory.
Sistem membaca baris Excel mulai dari baris ke-6 (baris 1-5 biasanya header/template).
Melakukan perulangan (loop). Membaca sel: Tipe (Q/A), Isi Teks, Kunci Benar/Salah, Tingkat Kesulitan.
Jika sel bertipe 'Q' (Question), sistem menyimpan sebagai soal baru ke database, lalu menahan ID soal tersebut.
Saat baris berikutnya di-loop dan bertuliskan 'A' (Answer/Jawaban), jawaban tersebut akan langsung dikaitkan (insert) ke ID Soal terakhir yang tersimpan sebelumnya.
Modul_import_word.php (Format Ms. Word/HTML):
Mengizinkan import langsung dari Microsoft Word yang di-Save As Web Page (HTML).
Karena Word HTML menyimpan gambar ke dalam folder terpisah (nama_file_files/), sistem akan meminta user mem-zip seluruh dokumen tersebut. Sistem mengekstrak berkas .zip, membaca struktur dokumen dengan DOM Parser (DOMNode), lalu otomatis mencocokkan teks berserta gambar (inline images) yang letaknya menyatu, langsung insert batch menjadi soal di CBT secara realtime.
5. Export/Import Bank Soal antar Server (Tool_exportimport_soal.php)
Dipakai ketika sekolah memiliki banyak server CBT, dan ingin mensinkronkan data antar komputer/server. Fungsi Utama & Logika:

exportsoal():
Melakukan Query kompleks. Menarik data Topik, Soal di dalam topik tersebut, beserta riwayat Kunci Jawaban.
Men-serialize susunan data tersebut ke format JSON dan menyimpannya menjadi file text.
Melakukan penyalinan (copy) seluruh gambar dan audio dari folder uploads/topik_[id] lalu digabungkan dengan file JSON tersebut, dibuatkan paket .zip atau konfigurasi mentah untuk diunduh.
importsoal():
Membaca paket yang diekspor dari server lain. Kalau struktur ID di database server tujuan bertabrakan (misal server tujuan sudah punya soal_id 100), sistem secara otomatis melakukan re-alokasi (A.I) ID yang baru tanpa merusak kaitannya dengan pilihan jawabannya.
6. File Manager (Modul_filemanager.php)
Layar manajemen khusus untuk admin/guru mengelola berkas media terpusat secara rahasia. Fungsi Utama & Logika:

tambah_dir(), upload_file(), hapus_file(): Berinteraksi langsung dengan sistem berkas (OS Filesystem) server PHP. User dapat membuat folder menggunakan perintah mkdir, upload menggunakan CodeIgniter Upload class (dengan limit tipe khusus gambar dan MP3), serta hapus file/folder menggunakan unlink() dan rmdir_recursive.
Direktori dasarnya berada di dalam public/uploads/.
Gambar dan Audio yang di-upload melalui File Manager ini dapat di-browse dan disisipkan saat admin menggunakan Text Editor Pilihan Ganda maupun Text Editor Soal.
