Blueprint Migrasi SAS CBT: PHP to Astro + Turso + Cloudflare
Blueprint ini menguraikan tahapan rekayasa ulang (rewrite) aplikasi CBT berbasis CodeIgniter/PHP menuju arsitektur Edge-Native modern menggunakan Astro JS (Fullstack SSR), Tailwind CSS, dan Turso (LibSQL) yang di-deploy di Cloudflare Pages.

Fase 1: Inisialisasi & Persiapan Arsitektur (Setup)
Fase ini bertujuan untuk meletakkan pondasi proyek, menyiapkan repositori, dan mengonfigurasi alat-alat modern.

Inisialisasi Proyek Astro
npm create astro@latest (Pilih mode Empty atau Blog sebagai dasar, tambahkan TypeScript).
Menambahkan integrasi Tailwind CSS: npx astro add tailwind.
Menambahkan integrasi Cloudflare Adapter: npx astro add cloudflare (Mengubah Astro menjadi mode Server-Side Rendering (SSR) yang diperlukan untuk API dan Session/Login).
Konfigurasi Tambahan Frontend
Instalasi library komponen UI (Rekomendasi: shadcn/ui atau Preline untuk komponen Tailwind siap pakai yang modern dan accessible).
Instalasi Alpine.js (@astrojs/alpinejs) atau React/Preact (@astrojs/preact) khusus untuk Interactive Islands (misal: Timer Ujian, Radio button jawaban Ajax).
Persiapan Database Turso & ORM
Buat database baru di turso.tech.
Instalasi ORM modern: npm i drizzle-orm @libsql/client dan npm i -D drizzle-kit.
Konfigurasi drizzle.config.ts untuk terhubung ke URL Turso menggunakan Auth Token.
Fase 2: Migrasi Skema Database & Autentikasi
Menerjemahkan tabel MySQL lama (19 tabel) ke dalam skema TypeScript Drizzle, dan membangun gerbang keamanan.

Definisi Skema Drizzle (src/db/schema.ts)
Terjemahkan tabel krusial terlebih dahulu: cbt_user, cbt_tes, cbt_soal, cbt_jawaban, cbt_tes_user, cbt_tes_soal.
Implementasikan relasi (One-to-Many, Many-to-Many) sesuai logika ERD aplikasi lama.
Push skema ke Turso: npx drizzle-kit push.
Membangun Sistem Autentikasi Modern (Session/JWT)
Implementasi autentikasi menggunakan Lucia Auth (sangat cocok untuk Astro + Drizzle) atau sistem JWT-Cookie kustom.
Buat API endpoint /api/auth/login dan /api/auth/logout.
Validasi Level Pengguna: Bedakan akses tipe admin, operator (pengawas), dan siswa.
Middleware Astro (src/middleware.ts)
Buat middleware global yang akan dieksekusi oleh Cloudflare Worker di setiap request.
Logika Gatekeeper: Cek validitas cookie session, proteksikan rute /admin/* hanya untuk admin/guru, dan rute /ujian/* hanya untuk siswa yang aktif.
Fase 3: Pembangunan Dashboard Admin (Backend UI)
Membangun ulang fungsionalitas CRUD manajerial dengan UI Tailwind yang responsive dan cepat.

Layout Utama Admin
Buat AdminLayout.astro dengan Sidebar navigasi (Menu Bank Soal, Tes, Laporan) dan Header (Profil, Logout).
Modul Manajemen Data Dasar
/admin/master: CRUD Mata Pelajaran, Topik, Grup/Kelas.
/admin/user: CRUD Data Siswa dan Data Administrator/Guru.
Modul Bank Soal (Crucial)
UI untuk membuat soal Pilihan Ganda, Esai, dan Jawaban Singkat.
Integrasi Editor WYSIWYG (seperti Quill.js atau TipTap) untuk menampung format soal kaya (gambar, teks tebal).
Membangun API Upload Media (Gambar/Audio) menggunakan R2 Cloudflare Object Storage (sebagai pengganti folder /uploads PHP).
Modul Manajemen Tes/Ujian
Formulir pembuatan Tes (Seting Waktu, KKM, Durasi, Acak Soal).
Distribusi Token: Endpoint API untuk generate Token aktif.
Fase 4: Pembangunan Portal Ujian Siswa (Mesin CBT)
Ini adalah core engine aplikasi, migrasi langsung dari Tes_kerjakan.php.

Dashboard Siswa & Persiapan Ujian
Halaman / setelah login: Menampilkan daftar tes yang sedang berjalan untuk Grup siswa tersebut.
Input Token: Konfirmasi sebelum memulai ujian.
Engine Pembuat Soal Acak (Randomization)
Saat siswa memvalidasi token, API Endpoint /api/ujian/mulai akan berjalan:
Baca parameter tes.
Tarik soal sesuai topik.
Acak urutan soal (menggunakan shuffle array di JS).
Acak opsi jawaban (A,B,C,D,E).
Simpan cetak biru (blueprint) acak ini ke tabel cbt_tes_soal di Turso untuk siswa tersebut.
Halaman Pengerjaan Interaktif (/ujian/[tes_id])
Menggunakan Preact/Alpine Island untuk SSR Hydration.
Timer Countdown: Logika waktu sisa yang dihitung dari Database Creation Time VS Current Date.
Navigasi Nomor Soal: Kotak navigasi (Navigasi Kiri/Kanan) dengan pewarnaan state (Belum dijawab abu, Sudah biru, Ragu kuning).
Auto-Save Jawaban via Fetch API
Setiap klik Radio Button / ketikan essay akan memicu fetch('/api/ujian/simpan') di peramban background.
API ini mem-verifikasi kecocokan kunci jawaban dan meng-update tabel nilai sementara Siswa di Turso.
Fase 5: Pelaporan & Monitoring (Analytics)
Live Monitoring Pengawas
Halaman /admin/pengawas yang menggunakan polling (misal setInterval setiap 10 detik memanggil API Astro) untuk meminta data siapa saja yang masih mengerjakan live.
Atau jika butuh real-time instan, bisa memanfaatkan Cloudflare Durable Objects atau WebSockets, namun polling Ajax sudah cukup mereplika versi CI.
Rekap Hasil Ujian
Kalkulasi akhir skor.
Ekspor ke Excel: Menggunakan library seperti exceljs di backend JS untuk mereplika PHPExcel lama, generate lalu download.
Analisis Butir Soal
Kuery agregasi dari Drizzle untuk menghitung Tingkat Kesulitan soal berdasar berapa persen siswa yang menjawab benar/salah.
Fase 6: Deployment & Production (Cloudflare Pages)
Koneksi Repositori GitHub
Buka Cloudflare Dashboard -> Pages -> Connect to Git.
Pilih repositori tempat kode Astro di-push.
Konfigurasi Build & Environment Variables
Build command: npm run build
Build output: dist
Set Environment Variables rahasia di Cloudflare: TURSO_DATABASE_URL, TURSO_AUTH_TOKEN, JWT_SECRET.
Setup R2 Object Storage (Untuk Media Upload)
Bind R2 Bucket ke Cloudflare Worker Pages Anda agar aplikasi bisa mengunggah dan menampilkan berkas gambar ujian skalabilitas Edge.
Testing & Go-Live
Lakukan Load Testing pada URL production akhir (https://ujian-sekolah.pages.dev).
Konfigurasi Custom Domain (missal: cbt.namasekolah.sch.id).
Sistem berjalan 100% pada Serverless Edge Environment.
