Penjelasan Detail Fitur: E. Pengaturan Sistem & Pengguna
Fitur ini merupakan wilayah manajerial tertinggi (Superadmin / Operator) yang memastikan operasional CBT berjalan lancar, akses pengguna terjaga secara aman, sistem dapat distel sesuai identitas sekolah, serta pengamanan ketersediaan data (backup).

1. Manajemen Pengguna (Useratur.php)
Fitur ini khusus dikembangkan untuk mengelola siapa saja (Admin, Guru, Proktor) yang diperbolehkan masuk ke dashboard/backend manajer aplikasi CBT (bukan siswa).

Logika & Fungsionalitas:

Sistem Akun (CRUD):
Controller ini menyimpan username, password, nama guru/admin, dan level akses (peran). Tambahan metadata ditaruh dalam field opsi1, opsi2, dan keterangan.
Enkripsi Keamanan Dasar:
Untuk pelindungan privasi dan integritas terhadap kebocoran database, password admin tidak disimpan dalam teks polos (plain text). Di fungsi add() dan edit(), kode melakukan hashing menggunakan algoritma sha1($password) sebelum dieksekusikan ke Model users_model->save().
Proteksi Hapus:
Terdapat mekanisme dasar yang memastikan bahwa identitas admin utama yang esensial tidak bisa ditimpa begitu saja secara sembarangan.
2. Hak Akses / Role-Based Access Control (Userlevel.php & Usermenu.php)
Aplikasi ini memiliki pilar otorisasi Role-Based Access Control (RBAC) yang sangat mumpuni. Ketimbang meng-hardcode satu per satu siapa yang berhak melihat apa, ia membuat sebuah matriks level dinamis secara relasional.

Logika & Fungsionalitas:

Penyusunan Menu (Usermenu.php):
Admin pusat bisa mendaftarkan URL menu-menu navigasi (contoh: menu Bank Soal, Menu Tes, Menu Laporan).
Setiap menu ditandai bertipe Parent (Tingkat Utama) atau Child (Sub-menu / Dropdown). Semua antarmuka kiri sidebar di-generate dari struktur data tabel ini.
Relasi Hak Akses & Matriks Level (Userlevel.php):
Controller ini membuat Peran/Level (misal: "Guru BK", "Guru Sejarah", "Operator Lab").
Inti kehebatannya terletak pada tabel user_akses. Saat Superadmin menambahkan level baru, ia dihadapkan pada ratusan Checkbox yang mewakili semua menu aplikasi.
Untuk setiap menu, Superadmin dapat menyetel Tiga Pilar Otorisasi Khusus:
Bisa Akses (Read / View)? -> Checkbox kodemenu[] aktif.
Bisa Tambah (Create)? -> Checkbox add[] aktif.
Bisa Ubah/Hapus (Update/Delete)? -> Checkbox edit[] aktif.
Ini memastikan bahwa "Guru Bahasa Indonesia" bisa Login, namun hanya dapat melihat matriks nilai, tanpa disugesti tombol Hapus Tes atau Edit Nilai.
3. Pengaturan Aplikasi (Pengaturan_zyacbt.php)
Memusatkan perubahan identitas teks front-end dan perilaku dasar aplikasi tanpa merubah satu sintaks pun di dalam file PHP. Secara langsung mengambil dan memperbarui tabel konfigurasi dasar.

Pengaturan Konfigurasi (Key-Value) yang Disimpan:

cbt_nama & cbt_keterangan: Judul raksasa yang menempel di Header Web dan Title Browser.
link_login_operator: Menyediakan kontrol untuk mengubah routing URL atau link navigasi tombol pendaftaran.
cbt_mobile_lock_xambro: Fitur pengamanan mutakhir. Jika dihidupkan, Siswa yang membuka ujian menggunakan HP Android / iOS Web biasa akan diblokir total secara otomatis oleh Javascript dan Controller Front-End, me-wajibkan siswa men-download APK "Mobile Exam Browser" khusus (Safe Exam Browser/SEB) agar tidak bisa membuka tab contekan (cheating_prevention).
proteksi_multilogin: Mengaktifkan session blocker untuk membentengi satu user-id ter-login dari 2 komputer barengan (menghindari penunggangan ujian / joki jarak jauh).
cbt_informasi: Papan pengumuman (marquee/dashboard alert) dinamis kepada siswa.
4. Backup Database & Pembersihan Sesi (Tool_backup.php)
Senjata Fail-Safe (Antisipasi Bencana) jika mendadak database rusak (mati lampu/hardisk jebol) atau butuh dipindah ke server lain. Dirancang kebal macet untuk data tes skala besar (ribuan siswa ribuan soal).

Logika & Fungsionalitas:

Durasi Paksa (Uncapped Memory):
Karena proses ekspor/zip menguras RAM Server yang besar jika ada puluhan ribu jawaban, kode diawali bypass server limit: ini_set("memory_limit","-1") & ini_set('max_execution_time', 200) detik.
Backup Database (database()):
Memakai Library CodeIgniter bawaan $this->dbutil->backup(), engine MySQL me-rontgen semua skema tabel ujian, menjadikannya satu tumpukan file .sql raksasa. Kemudian library helper force_download() melemparkannya ke perambah lokal dengan format .gz termampat (dikompresi sangat padat).
Backup Media (data_upload()):
Mengeksekusi Library Zip $this->zip->read_dir(), mengambil seluk-beluk seluruh file gambar, audio soal yang terunggah dalam /root/uploads/ lalu dipadatkan jadi satu file backup_data_upload_zya_cbt.zip.
Sapu Bersih Nyangkut / clear_session():
Pemusnah paksa massal jika Server Error. Memerintahkan cbt_sessions_model->empty_table(). Jika terjadi server bottleneck (siswa tidak bisa login meski komputer tak mati, token menolak jalan), tabel sesi Ci_sessions berbasis database ini akan langsung dibersihkan menjadi kosong (TRUNCATE), memaksa SEMUA user (admin, guru, proktor, siswa seantero sekolah) logout detik itu juga untuk meriset ulang traffic server dari Titik Nol.
